# Security policy

This repository is a synthetic-data portfolio demo. It is not a production system.

## Data

- All data in `sidley_pbip_spinup_package/output_test/.../data/` is **synthetic** and generated by `sidley_pbip_spinup_package/scripts/generate_sidley_pbip.py`.
- There is **no Sidley, client, matter, billing, financial, employee, or confidential data** in this repository.
- Public Sidley.com attorney names appear in `dim_attorney` only - see [`ATTORNEY_NAMES_ATTRIBUTION.md`](sidley_pbip_spinup_package/docs/ATTORNEY_NAMES_ATTRIBUTION.md).

## Secrets and credentials

- The Databricks integration reads tokens from environment variables (`DATABRICKS_HOST`, `DATABRICKS_TOKEN`, `DATABRICKS_WAREHOUSE_ID`).
- A local override file `sidley_pbip_spinup_package/scripts/databricks.config.json` is supported and is **gitignored**.
- `.env` and `.env.*` (except `.env.example`) are gitignored.
- **No credentials, tokens, or workspace URLs are committed in this repository.**

If you spot a leaked credential, file token, or anything that looks like real client data, please open a GitHub issue or email the repo owner so it can be rotated and force-removed from git history.

## Reporting

Open a GitHub issue with the label `security`, or email the repository owner.

There is no SLA - this is a personal portfolio repo. Responses are best-effort.

## Out of scope

- Vulnerabilities in upstream dependencies (`databricks-sdk`, `databricks-sql-connector`); please report those upstream.
- Anything depending on a hypothetical tenant deployment - this repo ships local CSV mode by default.
