# Previews

**These are previews, not Power BI screenshots.**

The PBIP report lives in `sidley_pbip_spinup_package/output_test/Sidley_BI_Modernization_Demo/`
and is intended to be opened in **Power BI Desktop**. The renderings in this folder are
honest, programmatic visualizations of the **same synthetic gold CSVs** the report binds
to, generated by Python scripts so that reviewers can see the data shape and layout
intent without launching Power BI.

## Why both formats

- **PNGs** (`pages/*.png`) - one per PBIP page. Easy to embed in GitHub markdown and on
  the repo landing page.
- **HTML mockup** (`dashboard_mockup.html`) - one self-contained file that aggregates
  the same numbers in a single dashboard-style layout. Open it in any browser; no
  build step required.

Both clearly carry a "synthetic data preview - not a Power BI screenshot" banner so
they cannot be misread as actual Power BI captures of real data.

## How to regenerate

```powershell
# One-time: install the rendering deps
pip install -r sidley_pbip_spinup_package\requirements-preview.txt

# 8 PNGs into previews/pages/
py sidley_pbip_spinup_package\scripts\render_page_previews.py

# Single HTML mockup into previews/
py sidley_pbip_spinup_package\scripts\build_html_mockup.py
```

Both scripts read from
`sidley_pbip_spinup_package/output_test/Sidley_BI_Modernization_Demo/data/*.csv`.
Regenerate that data first if you've changed the synthetic-data seed:

```powershell
py sidley_pbip_spinup_package\scripts\generate_sidley_pbip.py
```

## What the real Power BI report adds

Things these previews **cannot** show, but the PBIP build does:

- Cross-filtering, drillthrough, and bookmark navigation
- Slicer sync groups (`SidleySync_*`)
- Row-level security (RLS) roles: `Chicago Office Demo`, `OfficeKey One (DAX sample)`,
  `Finance Stakeholder`
- Time-intelligence calc group (`Current`, `MTD`, `QTD`, `YTD`, `PY`, `YoY`)
- Format strings on measures, certified extracts, and refresh observability tiles
- Live binding to a Databricks UC SQL Warehouse (toggle `pDataSource = "Databricks"`)

For all of that, open
`sidley_pbip_spinup_package/output_test/Sidley_BI_Modernization_Demo/Sidley_BI_Modernization_Demo.pbip`
in Power BI Desktop.
